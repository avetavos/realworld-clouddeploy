CloudDeploy is the **infrastructure capstone** of the series. You provision a production-shaped platform on **AWS, GCP, and Azure** with **Terraform** and **Terragrunt**, stand up managed Kubernetes on each (**EKS / GKE / AKS**), and deploy **ShopMicro** — the microservices app from project #3 — for real, wired to **Datadog** (observability), **Keycloak** (identity), and **GrowthBook** (feature flags), all shipped by **CI/CD**.

This introduction sets the scene before you write any HCL:

- [Overview](/clouddeploy/en/introduction/overview/) — what CloudDeploy is, the feature list, and the map from each module to the Learn Hub course it adapts.
- [Architecture](/clouddeploy/en/introduction/architecture/) — how the modules, the `live/` tree, the clusters, and the platform layer fit together, and why it's multi-cloud.
- [Prerequisites](/clouddeploy/en/introduction/prerequisites/) — the tools and cloud accounts to set up before Module 1.
- [Roadmap](/clouddeploy/en/introduction/roadmap/) — where CloudDeploy sits as the finale of the Real-World Projects series.
